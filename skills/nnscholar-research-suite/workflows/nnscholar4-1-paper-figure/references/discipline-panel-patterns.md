# Discipline Panel Patterns

Use this file when deciding panel order and grouping.

## Clinical Medicine

Common main figure:
- A: cohort / study design / data source
- B: model discrimination or primary outcome
- C: calibration or decision curve
- D: subgroup forest plot
- E: external validation or sensitivity analysis

Rules:
- Keep cohort flow and model performance visually separate.
- Do not mix internal and external validation without clear labels.

## Basic Biomedicine

Common main figure:
- A: mechanism or experimental design schematic
- B: representative image/gel/blot
- C: quantification plot
- D: perturbation/rescue result
- E: pathway or omics support

Rules:
- Pair representative image panels with quantification panels.
- Preserve scale bars and exposure/condition labels.

## AI / Data Science

Common main figure:
- A: model/pipeline architecture
- B: benchmark comparison
- C: ablation
- D: error analysis or calibration
- E: case visualization

Rules:
- Keep training/inference/evaluation panels distinct.
- Show dataset and metric labels consistently.

## Materials / Chemistry

Common main figure:
- A: synthesis or structure-function schematic
- B: SEM/TEM/AFM morphology image
- C: XRD/FTIR/Raman/UV-vis characterization
- D: performance curve
- E: stability/cycling/release/degradation

Rules:
- Preserve scale bars and instrument axes.
- Avoid decorative material icons that imply unsupported structure.

## Education / Psychology

Common main figure:
- A: intervention or conceptual framework
- B: main effect plot
- C: mediation/moderation or mechanism result
- D: heterogeneity/robustness
- E: attrition or measurement reliability if needed

Rules:
- Separate observed data from latent constructs.
- Do not imply causal pathways for correlational designs.

## Economics / Social Science

Common main figure:
- A: identification strategy or policy timeline
- B: descriptive trend / pre-trend
- C: main coefficient/event-study plot
- D: heterogeneity
- E: robustness/placebo

Rules:
- Keep identification logic and results distinct.
- Show uncertainty and pre-trends when relevant.

## Humanities

Common main figure:
- A: corpus/source construction
- B: analytical framework
- C: theme/entity/topic distribution
- D: comparative cases
- E: evidence map or interpretive argument structure

Rules:
- Prefer argument hierarchy over causal arrows unless explicitly causal.
- Label corpus selection and coding rules.

## Engineering

Common main figure:
- A: system/prototype architecture
- B: operating condition or experimental setup
- C: main performance curve
- D: stress/failure/safety test
- E: deployment or feedback loop

Rules:
- Label interfaces and operating conditions.
- Do not compare incompatible hardware/workloads in the same visual group.
