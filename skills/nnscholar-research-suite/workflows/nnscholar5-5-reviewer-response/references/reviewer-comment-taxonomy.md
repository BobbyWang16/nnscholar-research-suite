# Reviewer Comment Taxonomy

| Type | Meaning | Response strategy |
|---|---|---|
| Mandatory fix | Editor/reviewer requires change | Revise and cite location |
| Clarification | Reviewer did not understand text | Clarify manuscript and response |
| New analysis/experiment | Request for additional evidence | State completed work or explain limits |
| Citation/literature | Missing context or comparator | Add citations if valid |
| Formatting/reporting | Checklist, style, figure/table issue | Fix directly |
| Misunderstanding | Comment based on misread | Politely clarify and revise wording to prevent recurrence |
| Unreasonable/impossible | Cannot be done | Explain respectfully with evidence and limitation |
| Editor-only | Decision-level instruction | Address first in response letter |

## High-Impact Response Labels

Use these labels for Nature-family, Cell, Science, CNS-style, flagship society, or other high-impact revisions.

| Label family | Allowed labels | Use |
|---|---|---|
| Severity | minor / major / blocking / unclear | Estimate revision risk and whether author input or completed work is required |
| Accept and change | ACCEPT_TEXT / ACCEPT_ANALYSIS / ACCEPT_EXPERIMENT / ACCEPT_FIGURE | Use only when the change or artifact exists or is explicitly planned by the author |
| Clarify or constrain | CLARIFY_EXISTING / ADD_CITATION / SOFTEN_CLAIM / PARTIAL | Use when the safest response is precision, added support, or narrowed claims |
| Decline or block | DISAGREE / OUT_OF_SCOPE / AUTHOR_INPUT_NEEDED / BLOCKING | Use when the request cannot be satisfied without new evidence, author decision, or external material |

Never mark a response ready when it lacks a manuscript location, completed artifact, verified citation, or visible author-query placeholder for missing information.
